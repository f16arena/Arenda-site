# Changelog

Все заметные изменения сайта фиксируются здесь. Версии ведем в формате `MAJOR.MINOR.PATCH`.

## 1.3.15 - 2026-05-04

- На странице `/admin/requests` фильтры `Все`, `Новые`, `В работе`, `Выполнены` стали рабочими.
- В фильтрах показываются счетчики по статусам: новые, активные и закрытые заявки.
- Если выбранный фильтр пустой, экран предлагает вернуться ко всем заявкам, а не показывает тупиковое “нет данных”.
- Добавлена rollback-точка `rollback/pre-request-status-filters-1.3.15`.

## 1.3.14 - 2026-05-04

- Добавлен общий компонент пустого состояния с понятным объяснением и быстрыми действиями.
- Списки арендаторов, заявки, начисления, оплаты и расходы теперь показывают не просто “нет данных”, а что означает пустой экран и куда нажать дальше.
- В финансах исправлена нерабочая кнопка `Добавить` в блоке расходов: теперь там открывается настоящий диалог добавления расхода.
- Добавлена rollback-точка `rollback/pre-actionable-empty-states-1.3.14`.

## 1.3.13 - 2026-05-04

- Убран старый шум ESLint: проект теперь проходит `npm run lint` без предупреждений.
- Очищены неиспользуемые импорты, переменные и устаревшие eslint-disable в админке, API генерации документов, mobile shell, seed и служебном скрипте.
- Поведение экранов и бизнес-логика не менялись; это техническая чистка, чтобы новые проблемы в CI было видно сразу.
- Добавлена rollback-точка `rollback/pre-lint-cleanup-1.3.13`.

## 1.3.12 - 2026-05-04

- В `/admin/data-quality` добавлена критичная проверка некорректных контактов арендаторов: телефон должен быть казахстанским `+7`, email должен быть латиницей, с `@` и валидным доменом по синтаксису.
- Старые записи вроде `986`, `8983` или email с кавычками/кириллицей теперь выводятся отдельной проблемой с переходом в карточку арендатора.
- Проверка учитывает текущий режим здания: выбранное здание или все доступные здания.
- Добавлена rollback-точка `rollback/pre-contact-data-quality-1.3.12`.

## 1.3.11 - 2026-05-04

- В формах арендатора добавлена маска телефона Казахстана: поле автоматически держит префикс `+7` и не дает вводить буквы или номера не из диапазона KZ.
- Создание арендатора теперь требует валидный казахстанский телефон на сервере, даже если форму попытались обойти.
- Email арендатора теперь очищается от не-ASCII символов в UI и проверяется на сервере: латиница, обязательный `@`, домен и DNS-проверка почтового сервиса, когда DNS доступен.
- Импорт арендаторов также использует усиленную проверку email-домена и казахстанского номера.
- При редактировании контактов арендатора добавлена проверка дублей телефона/email до сохранения.
- Добавлена rollback-точка `rollback/pre-strict-contact-validation-1.3.11`.

## 1.3.10 - 2026-05-04

- На dashboard расширен блок `Сегодня`: добавлены оплаты арендаторов, ожидающие подтверждения администратором.
- На dashboard добавлен счетчик ошибок качества данных с переходом в `/admin/data-quality`.
- Операционные карточки dashboard теперь учитывают текущий режим здания: все доступные здания или выбранное здание.
- Исправлена ссылка карточки поступлений: теперь ведет в `/admin/finances`.
- Добавлена rollback-точка `rollback/pre-dashboard-action-center-1.3.10`.

## 1.3.9 - 2026-05-04

- FAQ переведен из статического массива в базу данных организации через таблицу `faq_articles`.
- При первом открытии FAQ организация автоматически получает копию базовых вопросов для владельца, администратора и арендатора.
- В `/admin/faq` добавлен редактор: можно создавать, менять, скрывать и снова включать вопросы, разделы, шаги, теги и ссылки.
- `/admin/faq` и `/cabinet/faq` теперь читают FAQ из БД, но сохраняют fallback на встроенные вопросы, если таблица еще недоступна.
- Добавлена миграция с RLS deny policy для `faq_articles` и rollback-точка `rollback/pre-db-faq-1.3.9`.

## 1.3.8 - 2026-05-04

- Добавлена таблица `payment_reports`: сообщение арендатора об оплате теперь хранится как финансовая заявка со статусом `PENDING`, `CONFIRMED` или `REJECTED`.
- Арендатор может приложить фото или PDF чека к форме `Я оплатил`; файл ограничен типами PDF/JPG/PNG/WebP и размером 1.5 МБ.
- В админских финансах появилась очередь оплат от арендаторов: администратор видит сумму, дату, назначение, комментарий и чек.
- Администратор может провести заявку в платеж, выбрать счет зачисления и закрыть связанные начисления.
- Администратор может отклонить заявку с причиной; арендатор получает уведомление.
- В чате теперь отображаются вложения, если сообщение содержит чек.
- Добавлена миграция с RLS deny policy для `payment_reports` и rollback-точка `rollback/pre-payment-report-review-1.3.8`.

## 1.3.7 - 2026-05-04

- Обновлен экран финансов арендатора: добавлен понятный платежный блок с реквизитами арендодателя, суммой долга, назначением платежа и QR с реквизитами.
- Убраны захардкоженные тестовые реквизиты из кабинета арендатора; используются реквизиты арендодателя из `lib/landlord.ts`.
- Телефон и email владельца по-прежнему не показываются арендатору; связь идет через администратора.
- Добавлена кнопка `Я оплатил`: арендатор отправляет администратору сообщение с суммой, датой, назначением платежа и комментарием/номером чека.
- В кабинете арендатора добавлена краткая история проведенных оплат.
- Добавлена rollback-точка `rollback/pre-tenant-payment-guide-1.3.7`.

## 1.3.5 - 2026-05-04

- Добавлены явные deny-by-default RLS policies для public-таблиц, где RLS был включен, но policies отсутствовали.
- Убраны grants для `anon` и `authenticated` на application tables в `public`, чтобы Supabase Data API не мог читать или менять tenant-данные напрямую.
- Добавлен revoke default privileges для будущих public-таблиц, созданных текущей DB-ролью.
- Проверка `/admin/system-health` теперь ловит public-таблицы с выключенным RLS, RLS-таблицы без policies и любые client grants для `anon/authenticated`.
- Добавлена rollback-точка `rollback/pre-public-rls-deny-policies-1.3.5`.

## 1.3.1 - 2026-05-03

- Добавлена страница `/admin/system-health` для production-проверки: env, база, таблицы, миграции, cron, email, документы, домен, sitemap/robots и журнал ошибок.
- `/api/health` переведен на общий актуальный health-check вместо старых SQL-migration проверок.

- Добавлена rollback-точка `rollback/pre-production-reliability-1.3.1`.

## 1.3.0 - 2026-05-03

- Добавлен режим владельца `Все здания`: дашборд, помещения, арендаторы, финансы, документы, календарь, аналитика и операционные страницы работают по всем доступным зданиям.
- Добавлен режим администратора `Мои здания`: один администратор может быть назначен на несколько зданий и видит только их данные.
- Добавлена таблица `user_building_access` и migration/backfill для существующих администраторов, бухгалтеров, завхозов и сотрудников.
- В управлении пользователями и сотрудниками добавлено обязательное назначение зданий для операционных ролей.
- Серверные actions дополнительно проверяют доступ к зданию/арендатору, чтобы нельзя было обойти UI подменой `buildingId`, `spaceId` или `tenantId`.
- Страница помещений в общем режиме показывает структуру `здание -> этаж -> помещение` и фактическую аренду занятых помещений через общий калькулятор.
- Финансы в общем режиме показывают данные по доступным зданиям и требуют выбрать здание при добавлении расхода.
- Добавлена rollback-точка `rollback/pre-multi-building-access-1.3.0`.

## 1.2.2 - 2026-05-03

- Кнопка `Очистить всё` в помещениях теперь открывает защитное окно с объяснением последствий.
- Для массовой очистки нужно вручную ввести слово `удалить`.

- Счетчик в предупреждении показывает все записи помещений, которые будут проверены перед удалением.
