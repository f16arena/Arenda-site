# Changelog

Все заметные изменения сайта фиксируются здесь. Версии ведем в формате `MAJOR.MINOR.PATCH`.

## 1.3.1 - 2026-05-03

- Добавлена страница `/admin/system-health` для production-проверки: env, база, таблицы, миграции, cron, email, документы, домен, sitemap/robots и журнал ошибок.
- `/api/health` переведен на общий актуальный health-check вместо старых SQL-migration проверок.
- Error boundaries теперь показывают пользователю короткий код `Ошибка #...`, а детали отправляют во внутренний endpoint `/api/errors/report`.
- Ошибки клиентского рендера пишутся в server log и `audit_logs`, чтобы разработчику было проще найти stack trace и контекст.
- Добавлен GitHub Actions CI: `npm ci`, `npx prisma validate`, `npm run lint`, `npm run build`.
- `.env.example` дополнен production-переменными `ROOT_HOST`, `RESEND_API_KEY`, `EMAIL_FROM`, `SENTRY_DSN`, `NEXT_PUBLIC_SENTRY_DSN`.
- Добавлена rollback-точка `rollback/pre-production-reliability-1.3.1`.

## 1.3.0 - 2026-05-03

- Добавлен режим владельца `Все здания`: дашборд, помещения, арендаторы, финансы, документы, календарь, аналитика и операционные страницы работают по всем доступным зданиям.
- Добавлен режим администратора `Мои здания`: один администратор может быть назначен на несколько зданий и видит только их данные.
- Добавлена таблица `user_building_access` и migration/backfill для существующих администраторов, бухгалтеров, завхозов и сотрудников.
- В управлении пользователями и сотрудниками добавлено обязательное назначение зданий для операционных ролей.
- Серверные actions дополнительно проверяют доступ к зданию/арендатору, чтобы нельзя было обойти UI подменой `buildingId`, `spaceId` или `tenantId`.
- Страница помещений в общем режиме показывает структуру `здание -> этаж -> помещение` и фактическую аренду занятых помещений через общий калькулятор.
- Финансы в общем режиме показывают данные по доступным зданиям и требуют выбрать здание при добавлении расхода.
- Добавлена rollback-точка `rollback/pre-multi-building-access-1.3.0`.

## 1.2.4 - 2026-05-03

- Страница арендаторов теперь учитывает активное здание: в БЦ F16 не показываются свободные помещения из Magic room.
- Создание арендатора передаёт контекст текущего здания и не позволяет выбрать помещение из другого здания через подмену формы.
- Карточка арендатора фильтрует свободные помещения и этажи для назначения по текущему зданию и org-scope.
- Назначение помещения на сервере блокирует переселение арендатора между разными зданиями без явного переключения контекста.
- Добавлена rollback-точка `rollback/pre-building-scoped-tenant-spaces-1.2.4`.

## 1.2.3 - 2026-05-03

- Удаление счётчика теперь транзакционно удаляет связанные показания, чтобы база не падала на foreign key.
- После удаления счётчика обновляются админский и арендаторский экран показаний.
- Общая кнопка удаления теперь корректно показывает ошибки, возвращённые server action через `{ error }`, вместо технического production-сообщения.
- Добавлена rollback-точка `rollback/pre-meter-delete-fix-1.2.3`.

## 1.2.2 - 2026-05-03

- Кнопка `Очистить всё` в помещениях теперь открывает защитное окно с объяснением последствий.
- Для массовой очистки нужно вручную ввести слово `удалить`.
- Server action очистки дополнительно требует контрольное слово, поэтому действие нельзя обойти простым вызовом из UI.
- Очистка остается заблокированной, если есть помещения, привязанные к арендаторам, или этаж сдан целиком.
- Счетчик в предупреждении показывает все записи помещений, которые будут проверены перед удалением.
- Добавлена rollback-точка `rollback/pre-clear-spaces-guard-1.2.2`.

## 1.2.1 - 2026-05-03

- Контакты владельца больше не показываются арендаторам в tenant-facing сценариях.
- Кабинет сообщений арендатора показывает только администратора здания или ADMIN'ов организации; OWNER исключен.
- Server action сообщений запрещает арендаторам писать владельцу или делать broadcast обходом UI.
- Заявки арендатора уведомляют администратора, а не владельца напрямую.
- Публичная страница бронирования показывает контакт администратора вместо контактов здания/владельца.
- Экран подписания договора редактирует старые контакты владельца в тексте как “связь через администратора”; новые договоры больше не добавляют телефон/email владельца в реквизиты.
- Добавлена rollback-точка `rollback/pre-owner-contact-privacy-1.2.1`.

## 1.2.0 - 2026-05-03

- Добавлен раздел `/admin/data-quality` — центр качества данных для владельца, администратора и поддержки.
- Проверяются критичные противоречия: два способа аренды одновременно, арендатор без контактов, помещение свободно при привязанном арендаторе.
- Добавлены предупреждения по операционным рискам: арендатор без помещения, занятое помещение без арендатора, отсутствие подписанного договора, договор без дат, счет без срока оплаты.
- Раздел добавлен в sidebar и быстрый поиск `Ctrl+K`.
- Добавлена rollback-точка `rollback/pre-data-quality-center-1.2.0`.

## 1.1.2 - 2026-05-03

- В условиях аренды добавлен выбор единственного способа расчета: ставка этажа, индивидуальная ставка за м² или фиксированная сумма в месяц.
- Серверные действия и импорт арендаторов запрещают одновременное заполнение индивидуальной ставки за м² и фиксированной аренды.
- Шаблон Excel-импорта уточняет, что колонки `Ставка ₸/м²` и `Фикс. аренда ₸/мес` взаимоисключающие.
- Добавлена проверка/нормализация существующих двойных значений: при наличии фиксированной аренды очищается индивидуальная ставка за м².
- Добавлена rollback-точка `rollback/pre-rent-mode-1.1.2`.

## 1.1.1 - 2026-05-03

- В списке помещений аренда занятых кабинетов теперь считается через общий калькулятор с учетом индивидуальной суммы аренды, аренды целого этажа и индивидуальной ставки за м².
- Для свободных помещений сумма в колонке `Аренда/мес` показывается как ориентировочная по ставке этажа.
- Добавлена rollback-точка `rollback/pre-space-rent-display-1.1.1`.

## 1.1.0 - 2026-05-03

- В карточку арендатора добавлен блок переменных дополнительных начислений: свет, вода, вывоз мусора, отопление, охрана, интернет и прочее.
- Поле суммы по услуге активно только после установки галочки.
- Сохранение создает новое начисление за период или обновляет уже созданное неоплаченное начисление того же типа.
- Добавлена rollback-точка `rollback/pre-service-charges-1.1.0`.
