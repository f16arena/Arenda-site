# Changelog

Все заметные изменения сайта фиксируются здесь. Версии ведем в формате `MAJOR.MINOR.PATCH`.

## 1.3.27 - 2026-05-04

- Добавлен готовый DOCX-шаблон договора аренды `docs/templates/commrent-rental-contract-template.docx` без данных конкретного арендатора.
- В шаблон внесены рабочие placeholder'ы Commrent для арендатора, арендодателя, основания действия, помещения, дат, ставки, суммы цифрами/прописью, пени и реквизитов.
- Проверено: система распознает 34 placeholder'а, `docxtemplater` заполняет файл без ошибки, Microsoft Word открывает шаблон и экспортирует его в PDF.

## 1.3.26 - 2026-05-04

- Главный dashboard владельца получил блок “Требует внимания сегодня” с приоритетами: просрочки, оплаты на проверке, качество данных и истекающие договоры.
- Разрез по зданиям теперь сравнивает доход, расход, прибыль, долг, свободную площадь и заполняемость; добавлены экспорт Excel и страница печати/PDF для отчета собственника.
- Генератор договора при активном DOCX/XLSX-шаблоне больше не показывает старый договор из кода на странице создания: отображается активный загруженный шаблон и статус автоподстановки.
- Если загруженный DOCX/XLSX/PDF не содержит меток, система скачивает именно этот файл как статичный шаблон, а не подставляет старый стандартный договор.
- Поиск меток в DOCX стал надежнее: учитываются document/header/footer XML и метки, которые Word разбил на несколько внутренних тегов.
- Кнопка замены активного шаблона стала компактной; большой dropzone показывается только когда шаблон еще не загружен.
- Добавлена rollback-точка `rollback/pre-template-dashboard-1.3.26`.

## 1.3.25 - 2026-05-04

- Раздел `/admin/documents` теперь показывает отдельный рабочий блок “Создать документ” с выбором: договор, счёт на оплату, АВР и акт сверки.
- Добавлена страница `/admin/documents/new` и новые прямые пути создания: `/contract`, `/invoice`, `/act`, `/reconciliation`.
- Управление шаблонами документов перенесено в настройки: `/admin/settings/document-templates`; старый `/admin/documents/templates` мягко перенаправляет туда.
- Загрузка/удаление шаблонов больше не показывается внутри генераторов документов, чтобы создание документа и настройка шаблонов не смешивались.
- Обновлены sidebar, быстрый поиск, карточка арендатора, список договоров и Data Quality Center под новые пути.
- Добавлена rollback-точка `rollback/pre-document-create-settings-1.3.25`.

## 1.3.24 - 2026-05-04

- Генератор договора аренды теперь подставляет основание действия арендатора: для ИП — уведомление о начале деятельности, для ТОО/АО — Устав, для ЧСИ — лицензионные/регистрационные документы.
- Суммы аренды в договорах и шаблонах формируются в формате `75 000,00 (семьдесят пять тысяч)`.
- Обновлены пункты договора: отопление, электроэнергия и вода указаны как предоставляемые в рамках эксплуатации здания, а телефон/интернет и прочие услуги — как дополнительные; камера и вывеска размещаются только по согласованию сторон.
- В шаблоны договоров добавлены новые placeholder'ы: `tenant_basis`, `landlord_basis`, `monthly_rent_with_words`, `utilities_clause`, `signage_clause`.
- Загрузка DOCX/XLSX-шаблона без placeholder'ов теперь явно предупреждает, что такой файл останется статичным и не сможет автозаполнить арендатора, суммы и реквизиты.
- Добавлена rollback-точка `rollback/pre-contract-template-data-1.3.24`.

## 1.3.23 - 2026-05-04

- Расширены типы audit-событий: `ERROR`, `SECURITY`, `system`, `apiKey`.
- Добавлена rollback-точка `rollback/pre-quality-health-security-1.3.23`.

## 1.3.22 - 2026-05-04

- `/admin/requests` больше не загружает все заявки сразу: фильтры считаются через `groupBy`, список открывается страницами по 30 записей.
- `/admin/tasks` переведен на такой же режим: `count/groupBy/take/skip` вместо фильтрации полного списка в памяти.
- `/admin/finances` ограничивает видимые начисления и расходы страницами по 10 записей, а итоговые суммы месяца считает через Prisma `aggregate`, чтобы карточки финансов оставались точными.
- Добавлен общий компонент пагинации и helper для нормализации страниц.
- `npm run perf:audit` теперь проверяет бюджет размера файлов: client-файлы до 140 KB, server-файлы до 80 KB.
- Добавлена rollback-точка `rollback/pre-pagination-performance-1.3.22`.

## 1.3.21 - 2026-05-04

- `/admin/spaces` в режиме всех зданий больше не рендерит детальные таблицы всех помещений сразу: вместо этого показывается легкая сводка по зданиям и этажам, а полная таблица открывается после выбора конкретного здания.
- Визуальная карта этажа вынесена в отдельный lazy client-chunk, поэтому список помещений не тянет интерактивную SVG-карту в начальную загрузку.
- Карточка арендатора лениво догружает тяжелые client-секции: реквизиты, условия аренды, доп. начисления, документы, email-лог и назначение целого этажа.
- В карточке арендатора список свободных помещений ограничен первыми 50 записями с предупреждением, чтобы большие здания не замедляли открытие страницы.
- Dashboard владельца считает разрез по зданиям через `groupBy`, не загружая все помещения в память для подсчета заполняемости.
- Кабинет арендатора больше не тянет все неоплаченные начисления в основной tenant-запрос: итог долга считается агрегатом, список ограничен ближайшими начислениями, календарь платежей грузится лениво.
- Добавлена rollback-точка `rollback/pre-deep-page-performance-1.3.21`.
