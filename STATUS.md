# ArendaPro — статус проекта и план улучшений

> **F16 Arena** — система управления арендой бизнес-центра (3 этажа + подвал, ~3200 м²)
> Дата отчёта: апрель 2026

---

## 1. Технологический стек

| Слой | Технология |
|------|-----------|
| Framework | Next.js 16.2.4 (App Router, Server Components, Server Actions) |
| Runtime | React 19.2 |
| Auth | NextAuth v5 (beta), Credentials provider, JWT strategy |
| ORM | Prisma 7.8 + `@prisma/adapter-pg` |
| База данных | Supabase PostgreSQL (Transaction Pooler) |
| Стили | TailwindCSS 4, lucide-react |
| Валидация | Zod 4 |
| Хостинг | Vercel |
| Хеширование паролей | bcryptjs |

---

## 3. Что нужно улучшить

### 3.1 Безопасность (приоритет: высокий)

- [ ] **Пароли по умолчанию** — заменить на принудительную смену при первом входе
- [ ] **Rate limiting** на `/api/setup`, `/login` — защита от brute-force
- [ ] **CSRF-защита** — проверить что NextAuth её обеспечивает на server actions
- [ ] **2FA / OTP по SMS** — особенно для ролей OWNER / ACCOUNTANT
- [ ] **Логирование действий** — audit log: кто, когда, что изменил
- [ ] **Row Level Security (RLS)** в Supabase — сейчас отключён, но надо вернуть после переезда на Supabase Auth
- [ ] **Не показывать тестовые пароли** на странице логина в продакшене
- [ ] **Удалить `/api/setup` после инициализации** или прятать за feature flag
- [ ] **Хранение секретов** — перенести `SETUP_SECRET` в Vault/Secrets Manager

### 3.2 Архитектура и качество кода

- [ ] **Юнит-тесты** — Vitest для actions, бизнес-логики (расчёт пеней, начисление аренды)

- [ ] **Middleware для авторизации** — централизовать проверки прав вместо повторяющихся `auth()` в каждой странице

## 4. Технический долг

| Что | Где | Почему чинить |
|-----|-----|--------------|
| Тестовые пароли в UI | `app/login/page.tsx` (hint block) | Не должно быть в продакшене |
| `/api/setup` доступен | `app/api/setup/route.ts` | Защищён только секретом, но эндпоинт есть |
| Нет `loading.tsx` | Все `app/admin/*` | Пустая страница при загрузке |
| `confirm()` вместо Dialog | space-actions, tenant-actions | Не выглядит профессионально |
| `console.error` без транспорта | `auth.ts` | Логи только в Vercel functions |
| `dev.db` в репозитории | корень | SQLite-артефакт от старой БД, удалить |
| Нет `.env.example` | корень | Новый разработчик не знает что задавать |
| Нет CHANGELOG.md | корень | История версий не ведётся |
| `as any` | `app/actions/auth.ts:36` | Type-cast для NextAuth error cause |
