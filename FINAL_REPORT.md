## 🟠 Фаза B — Авто-документы

### DOCX-генераторы
- ✅ **Договор аренды** — `/api/contracts/generate?tenantId=xxx&number=xxx`
  - Реквизиты Жампеисова Б.К. подставляются автоматически
  - Реквизиты арендатора (название, ИИН/БИН, адрес, директор, банк)
  - Расчёт суммы аренды (помещение × ставка ИЛИ фикс. этаж)
  - Имя файла: `Договор_F16-2026-007_Tenant.docx`

- ✅ **Счёт-фактура** — `/api/invoices/generate?tenantId=xxx&period=2026-04`
  - Таблица услуг (аренда + уборка)
  - Реквизиты обеих сторон
  - Итоговая сумма прописью

- ✅ **Акт оказанных услуг** — `/api/acts/generate?tenantId=xxx&period=2026-04`
  - Перечень услуг с периодом
  - Подписи и М.П. сторон

- ✅ **Акт приёма-передачи** — `/api/handover/generate?tenantId=xxx&direction=in|out`
  - Адрес, площадь, состояние помещения
  - in = приём арендатором, out = возврат арендодателю

### Авто-нумерация договоров
- ✅ Префикс задаётся для каждого здания (`Building.contractPrefix`)
- ✅ Формат: `{ПРЕФИКС}-{ГОД}-{№3-цифры}` → `F16-2026-001`
- ✅ Уникальный в пределах здания
- ✅ В UI поле с авто-предложением + ручной override

### Email-рассылка
- ✅ **Resend** интеграция (бесплатный тариф 100 писем/день)
- ✅ Брендированный HTML-шаблон с шапкой «БЦ F16»
- ✅ DOCX как вложение
- ✅ Кнопки **DOCX / Email** в карточке арендатора для каждого типа документа
- ✅ Если `RESEND_API_KEY` не задан — DOCX скачивается, email пропускается

### Tracking pixel
- ✅ 1×1 невидимый пиксель в HTML письма
- ✅ При открытии — запись в `email_logs.opened_at`
- ✅ Журнал писем на странице арендатора:
  - Тема, статус (отправлено / прочитано / ошибка)
  - Время отправки и открытия + счётчик открытий

### Cron — авто-начисления 1 числа
- ✅ Запускается каждое 1 число в 9:00 Алматы (Vercel Cron)
- ✅ `/api/cron/monthly-invoices`:
  - Создаёт `RENT` начисления для всех активных арендаторов
  - + `CLEANING` если включена опция
  - Уведомление в-приложение + Telegram

## 🟡 Фаза C — Финансовая автоматизация

### Реализовано
- ✅ **Авто-расчёт пеней** в существующем cron `/api/cron/check-deadlines`
  - Формула: `сумма × penaltyPercent% × дни_просрочки`, не более 10%
  - Заменяет вчерашнюю пеню на кумулятивную
  - Льготный период 1 день

- ✅ **Cashflow на дашборде**
  - 12 месяцев: 6 прошлых факт + 6 будущих прогноз
  - Параллельные запросы за все периоды
  - Зелёные/красные столбики с пунктиром для прогноза
  - Header: общий доход / расход / маржа

- ✅ **Excel-экспорт** `/api/export/finances`
  - 4 листа: Начисления / Платежи / Расходы / Сводка
  - Денежный формат (`#,##0 ₸`)
  - Цветные итоговые строки
  - Можно фильтровать по дате через `?from=&to=`

- ✅ **Audit log** — `/admin/audit` (только OWNER)
  - Записывает CREATE/UPDATE/DELETE/LOGIN/LOGOUT
  - Кто (имя + роль), что, когда, откуда (IP)
  - Helper `lib/audit.ts` для интеграции в любой action

- ✅ **Импорт банковской выписки** — `/admin/finances/import`
  - Парсер CSV из Kaspi Business / Halyk Online
  - Автоопределение разделителя (`,` `;` `\t`)
  - Поиск БИН/ИИН в назначении платежа
  - Авто-матч по: БИН → ИИН → название компании
  - Ручной override из dropdown
  - Apply → создаёт Payment записи

1. `001_create_schema.sql` — основные таблицы
2. `002_seed_users.sql` — стартовые пользователи
3. `003_rls_policies.sql` — отключение RLS
4. `004_tariffs_and_tenant_fields.sql` — тарифы, ИИН, full-floor аренда
5. `005_notifications_and_deadlines.sql` — уведомления, Telegram
6. `006_role_permissions.sql` — динамические права
7. `007_contract_numbering.sql` — авто-нумерация договоров
8. `008_email_logs.sql` — журнал писем
9. `009_audit_and_leads.sql` — audit log + CRM лиды

### Документация
- `STATUS.md` — статус реализации
- `ROADMAP.md` — план развития
- `COMPETITIVE_ANALYSIS.md` — анализ конкурентов
- `FINAL_REPORT.md` — этот документ

### API endpoints
**Документы:**
- `GET /api/contracts/generate` — DOCX договор
- `GET /api/invoices/generate` — DOCX счёт-фактура
- `GET /api/acts/generate` — DOCX акт услуг
- `GET /api/handover/generate` — DOCX акт приёма-передачи

**Финансы:**
- `GET /api/export/finances` — Excel (4 листа)
- `GET /api/export/1c` — 1C Enterprise format

**Cron:**
- `GET /api/cron/check-deadlines` — ежедневно 11:00, дедлайны + пени
- `GET /api/cron/monthly-invoices` — 1 числа 9:00, авто-начисления

**Интеграции:**
- `GET /api/egov/bin?bin=XXX` — проверка БИН через stat.gov.kz
- `GET /api/search?q=XXX` — глобальный поиск
- `GET /api/health` — диагностика БД

**Telegram:**
- `POST /api/telegram/webhook` — приём команд
- `GET /api/telegram/setup` — регистрация webhook

**Email:**
- `GET /api/email/track?id=XXX` — tracking pixel

### Cron-расписание (Vercel)
```json
{
  "crons": [
    { "path": "/api/cron/check-deadlines",    "schedule": "0 5 * * *" },
    { "path": "/api/cron/monthly-invoices",   "schedule": "0 4 1 * *" }
  ]
}
```
