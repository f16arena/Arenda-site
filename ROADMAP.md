# ArendaPro — Дорожная карта развития

> Что есть сейчас (после последнего раунда):
> - Авторизация, роли, супер-админ панель
> - CRUD арендаторов (с ИИН, реквизитами, документами по типу юрлица)
> - Помещения, этажи, аренда этажа целиком
> - Финансы (начисления, платежи, расходы, пени)
> - Счётчики с тарифами и авто-начислениями
> - Договоры — список + предупреждения за 20 дней + cron
> - Чат сообщений (личные + объявление всем)
> - Уведомления (in-app колокольчик + Telegram бот опционально)
> - Шаблон договора F16 Arena с автозаполнением → печать в PDF

## 🔴 Фаза 1 — Срочные доработки (1-3 дня каждая)

| Что | Зачем | Как |
|-----|-------|-----|
| Telegram бот настроить | Бесплатные SMS — самый частый канал в РК | 1) Создать бота через @BotFather 2) В Vercel → env: `TELEGRAM_BOT_TOKEN`, `NEXT_PUBLIC_TELEGRAM_BOT_NAME` 3) Каждый юзер: профиль → подключить |
| Vercel CRON_SECRET | Чтобы cron на /api/cron/check-deadlines работал и был защищён | Vercel → env: `CRON_SECRET=любая-длинная-строка` |
| Скрыть ссылки невидимые ролям | Бухгалтер не должен видеть «Помещения» и т.д. | Filter в admin-sidebar по `userRole` (как сейчас для OWNER) |
| Убрать `/api/setup` из прода | Безопасность — после инициализации не нужен | Удалить роут или защитить env-флагом `SETUP_ENABLED=false` |
| Документация для администратора | Пошаговая инструкция как пользоваться | Markdown в /admin/help |

## 🟠 Фаза 2 — Автоматизация финансов (1-2 недели)

| Что | Зачем | Как |
|-----|-------|-----|
| Авто-начисления аренды по cron | Сейчас вручную, нужно автоматом 1 числа | `/api/cron/generate-charges` + Vercel Cron `0 0 1 * *` |
| Авто-расчёт пеней | Сейчас руками, должна быть формула 1% × дни просрочки × сумма | Расширить cron: считать долги > paymentDueDay |
| PDF счёта-фактуры | Бухгалтерии нужны бумаги | `react-pdf` или браузерная печать как у договора |
| Excel-экспорт начислений и платежей | Для 1С и отчётности | библиотека `exceljs` + endpoint /api/export/finances |
| Журнал операций (audit log) | Кто, когда, что менял | Новая модель AuditLog, middleware на actions |
| Привязка платежа к нескольким начислениям | Сейчас платёж — одно начисление, в реальности один платёж может закрыть несколько | Перевести на n-к-n или поле `chargeIds` |
