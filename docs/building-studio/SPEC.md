# 1. РОЛЬ И КОНТЕКСТ

Principal Game Engine Architect + Senior Frontend + 3D WebGL Developer + UX Director + Product Designer.
Проектируем и реализуем **Commrent Building Studio** — модуль 3D-моделирования зданий внутри SaaS Commrent (commrent.kz, аренда коммерческой недвижимости, Казахстан).

Это НЕ MVP, НЕ демо, НЕ «кубики на Three.js», НЕ скучный CAD. Это красивый, плавный, дорогой, игровой Build Mode и одновременно корректная параметрическая система на 5+ лет. Упрощения архитектуры недопустимы; допустима только декомпозиция на модули при зафиксированной заранее архитектуре, каждый модуль — production-ready.

Не копировать чужие ассеты/лого/иконки/арт/интерфейс один в один. Продукт оригинальный, уровень игрового редактора строительства.

Бизнес-контекст: пользователь Commrent (владелец/управляющий ТЦ, БЦ, компьютерного клуба, кафе, офиса) строит цифровую модель объекта. Помещения модели связываются с сущностью `premise` в базе Commrent и подсвечиваются статусом аренды. Применение: (а) витрина для арендаторов, (б) операционная карта объекта, (в) планирование перепланировок.

При открытии страницы пользователь сразу понимает: «Да, здесь можно построить здание как в игре».

# 2. ЦЕЛЬ ПРОДУКТА

Страница `/builder`, где пользователь без CAD-подготовки (уровень игрока Sims 4):
- строит здание с нуля: фундамент, цоколь, любое число этажей, технический этаж, эксплуатируемая кровля, подземный паркинг;
- рисует стены, система сама замыкает помещения, строит полы и перекрытия;
- ставит окна/двери — проёмы вырезаются автоматически;
- строит лестницы, соединяющие этажи;
- генерирует крыши параметрически (плоская, двускатная, вальмовая, четырёхскатная, многоуровневая);
- редактирует рельеф участка, кладёт дороги, парковку, дорожки, озеленение, заборы, фонари;
- расставляет мебель/оборудование из каталога (GLB);
- красит стены/полы/фасады «ведром» как в Sims;
- связывает комнаты с помещениями Commrent, видит статусы аренды поверх 3D;
- сохраняет проект, делится read-only ссылкой (витрина);
- генерирует здание текстовым запросом (AI Mode).

Ориентиры: Sims 4 Build Mode + SketchUp + Planner5D/Floorplanner + premium real estate configurator.

# 4. АРХИТЕКТУРНЫЙ ФУНДАМЕНТ

## 4.1 Документ ≠ Геометрия
Проект — параметрический JSON-документ, единственный источник правды. Геометрия выводится детерминированной `buildGeometry(document) → meshes`. Меши не сериализуются. Изменение = мутация документа через команду → инкрементальная перегенерация затронутых мешей (dirty-flag по этажу/комнате, не полный rebuild).

## 4.2 Модель данных (ядро)
```
Project
└─ Site (terrain heightmap, дороги, парковка, ландшафт, заборы, освещение)
   └─ Building (1..n)
      └─ Floor (level: -2,-1,0=цоколь,1,2,...,tech,roof; elevation, height, visible, locked, opacity)
         ├─ WallGraph        — узлы (Vec2) + рёбра (стены) на плоскости этажа
         ├─ Rooms[]          — ПРОИЗВОДНЫЕ: грани планарного графа стен
         ├─ Openings[]       — (wallId, type, width, height, sillHeight, offset)
         ├─ Stairs[]         — параметрические, floorId → floorId, вырез в перекрытии
         ├─ Columns[]
         ├─ Objects[]        — (assetId, transform, attachTo, locked)
         └─ CeilingConfig
      └─ Roof (тип, footprint, уклон pitch, свес overhang, материал)
```
Ключевое: стена — НЕ пара точек start–end, а ребро графа с общими узлами. Подвинул узел — все примыкающие стены двигаются вместе. Обязательно.

Единицы: миллиметры в документе, метры в рендере (1 unit = 1 m). Сетка: 100 мм (точно) / 500 мм (быстро), угловые привязки 15°/45°/90°, инференс к узлам/рёбрам/перпендикулярам как в SketchUp. Ввод размеров с клавиатуры при рисовании: тянешь стену → печатаешь `4500` → Enter.

## 4.3 Геометрические алгоритмы (ядро, покрывается тестами)
1. Вставка стены: разбиение пересечений (segment intersection → split рёбер), слияние коллинеарных, snap к узлам в радиусе допуска.
2. Детекция помещений: грани планарного графа (half-edge обход по минимальному углу). Каждая внутренняя грань = Room с полигоном. Создаются/удаляются автоматически. Площадь из полигона.
3. Стыки: miter-соединение контуров по толщине (offset центральной линии, пересечение offset-линий в узлах). T- и X-стыки корректны.
4. Проёмы: профиль стены — extruded polygon с уже исключёнными прямоугольниками проёмов (основной путь); CSG2 — fallback. Drag вдоль стены, валидация. Окно/дверь НИКОГДА не «поверх» сплошной стены.
5. Полы: триангулированный полигон комнаты (earcut с дырками под лестничные вырезы). Перекрытие = пол верхнего этажа толщиной (по умолч. 300 мм).
6. Крыши: параметрический генератор. Вход — контур, тип, уклон, свес, высота конька. Скатные — straight skeleton, корректно на невыпуклых. Эксплуатируемая = плоская + флаг «является этажом».
7. Лестницы: прямая / Г / П / винтовая. Автоподбор ступеней (проступь/подступёнок 280–300 / 150–180 мм), перила toggle. Установка: связывает этажи, режет проём в перекрытии, валидирует габарит.
8. Terrain: heightmap-грид (1 м/ячейка, настраиваемо). Кисти: поднять/опустить/сгладить/выровнять (радиус, сила). Под пятном здания — building pad. Текстуры — splat map + triplanar на склонах.
9. Дороги/дорожки: сплайны с шириной → ribbon-меш по terrain, бордюры. Парковка — разметка-штампы вдоль ребра дороги (N × 2.5×5.3 м). Заборы — сплайн с тайлингом секций.

## 6.2 Команды и Undo/Redo
Все мутации документа — только Command Pattern: `{ apply(doc), revert(doc), merge?(next) }`. НЕ снапшоты всего проекта. История ≥ 200 шагов, drag = одна команда через merge. Команды — транспорт для AI Mode и будущей коллаборации.

## 6.3 Babylon-гигиена (обязательно)
Единственный Engine на страницу, защита от дублирования при re-render/StrictMode; корректный dispose сцены/материалов/текстур/observers при unmount; resize handling; защита от отсутствия canvas; typed guards; отсутствие утечек.

## 6.4 Производительность (бюджеты)
- 60 FPS: 10 000 объектов, 1 000 помещений, 30 этажей (референс: Apple M1 / Iris Xe)
- ≤ 300 draw calls (thin instances, merge статичной геометрии по этажу/чанку)
- Перегенерация при правке стены ≤ 16 мс на этаж; тяжёлое — в Worker
- LOD объектов каталога (3 уровня), frustum culling, выгрузка неактивных этажей
- Загрузка проекта на 1 000 помещений ≤ 3 с до интерактивности, геометрия прогрессивно

## 6.5 Камеры

Каждый инструмент — полупрозрачный, подсвеченный, плавный preview. Любое действие — мгновенный визуальный отклик.

# 10. AI MODE
Текст → здание:
1. Запрос → Anthropic API с системным промптом-генератором.
2. Модель возвращает строго JSON по Zod-схеме `BuildingSpec`: этажи, контур, сетка помещений, фасад, крыша, парковка. Без прозы.
3. Валидация Zod → компиляция `BuildingSpec` в последовательность обычных команд документа → применение одной undo-группой.
4. Результат полностью редактируем теми же инструментами.
Пример: «Создай трёхэтажное коммерческое здание с цоколем, парковкой на 20 мест, стеклянным фасадом и плоской кровлей» → корректный редактируемый объект.
