## 📋 Большое ТЗ — что внедряем по фазам

### 🔴 ФАЗА A — Графический план этажа (3-4 недели)

> Это главное, чем нас будут оценивать и почему вы об этом 3 раза просили

**A.1. Базовый редактор плана**
- SVG-холст с zoom (Ctrl+wheel) и pan (drag)
- Инструменты: прямоугольник, многоугольник (для диагональных стен), линия (для дверей/окон)
- Сетка с привязкой (snap to grid)
- Размеры в метрах (масштаб конфигурируемый)
- Сохранение в `Floor.layoutJson` (уже есть поле в БД)

**A.2. Интерактивный режим просмотра** (как на скриншоте Renda)
- Клик по помещению → попап с инфо: арендатор, договор, тариф, оплачено до, долг
- Цветовое кодирование:
  - 🟢 Свободно
  - 🔵 Занято
  - 🟡 Долг
  - 🔴 Просрочка > 10 дней
  - ⚫ Обслуживание
- Hover показывает мини-tooltip
- Режимы отображения: 100% / 75% / контур / без заливки

**A.3. Детали плана**
- Двери (схематичные, со стороны открывания)
- Окна на стенах
- Коридор / стены
- Подсказки этажа (метраж в углу комнаты)
- Текстовые подписи (например «Лестница», «Туалет»)

**A.4. Подложка (картинка плана)**
- Загрузить PNG/JPG плана здания (Supabase Storage)
- Режим прозрачности подложки
- Калибровка: «эта линия = 5 метров»

**A.5. 2D / 3D переключатель** (wow-фактор)
- 2D — классический план сверху
- 3D — изометрия с высотой стен
- Использовать `three.js` или `@react-three/fiber`

**Тех. стек для редактора:**
- `react-konva` (Canvas, проще SVG для сложных интерактивов) ИЛИ нативный SVG
- `zustand` для стейта редактора
- `@react-three/fiber` + `drei` для 3D

### 🟠 ФАЗА B — Автогенерация документов (2 недели)

**B.1. DOCX-генерация договоров**
- Библиотека `docxtemplater` + готовый шаблон с placeholders `{tenant.companyName}` и т.д.
- Сохранение в Supabase Storage
- Кнопка «Скачать DOCX» рядом с «Печать PDF»

**B.2. Авто-рассылка счёта-фактуры**
- Cron 1-го числа каждого месяца → создать начисление + сгенерировать счёт-фактуру PDF + отправить email
- Email через Resend (free tier 100/день)

**B.3. Акт оказанных услуг**
- Шаблон акта (как договор)
- При оплате → авто-генерация акта → отправка

**B.4. Шаблоны других документов**
- Доп. соглашение об изменении ставки
- Уведомление о повышении тарифа
- Акт приёма-передачи помещения
- Все по образцу: HTML preview → DOCX export → PDF print

**B.5. Отслеживание открытия писем**
- Pixel-tracking в email через Resend webhooks
- В карточке арендатора: «Счёт прочитан 15.04 в 14:23»

### 🟡 ФАЗА C — Финансовая автоматизация (2 недели)

**C.1. Импорт банковской выписки**
- Загрузка CSV из Kaspi Business / Halyk
- Авто-сопоставление платежей с арендаторами по БИН/ИИН
- Ручная разнеска нераспознанных
- Авто-закрытие начислений

**C.2. Перевыставление коммунальных по площади**
- Сейчас только по счётчику
- Добавить режим: общий расход здания / общая площадь = ₸/м²
- Распределение пропорционально арендатору

**C.3. Audit log (журнал операций)**
- Кто что менял, когда
- Просмотр в /admin/users/[id]/audit
- Защита от «никто не помнит кто удалил»

**C.4. Прогноз cashflow на 12 месяцев**
- На основе: активные договоры × ставка × месяцы до окончания
- График Recharts
- KPI «доход / расход / маржа»

**C.5. Авто-расчёт пеней**
- Cron ежедневно: просрочка > 0 → начислить пеню
- Формула: `остаток_долга × penalty_percent% × дни_просрочки`
- Создание `Charge` типа PENALTY автоматически

### 🟢 ФАЗА D — Лиды и CRM (1 неделя)

**D.1. Воронка продаж**
- Модель `Lead`: имя, контакт, требуемая площадь, бюджет, статус (NEW / SHOWN / NEGOTIATION / SIGNED / LOST)
- Канбан-доска
- Привязка к свободным помещениям

**D.2. Бронирование помещения**
- Lead может «забронировать» помещение на N дней
- Помещение помечается как RESERVED (новый статус)
- Авто-снятие брони если не подписали

**D.3. Источники лидов**
- Откуда узнал: сайт / сарафанное / Krisha / Авито
- Аналитика конверсии

### 🔵 ФАЗА E — Mobile, PWA, UX (2 недели)

**E.1. PWA (Progressive Web App)**
- `next-pwa` или ручной service worker
- Установка на главный экран телефона
- Работа offline (просмотр данных)
- Push-уведомления через Web Push API

**E.2. Мобильная адаптация**
- Все таблицы → cards на mobile
- Sidebar → drawer (выезжает по кнопке)
- Touch-friendly кнопки
- Быстрая подача показаний с телефона арендатором

**E.3. Тёмная тема**
- next-themes
- Палитра slate dark

**E.4. Глобальный поиск Cmd+K**
- `cmdk` библиотека
- Поиск по: арендаторам, помещениям, договорам, заявкам, документам
- Endpoint `/api/search?q=...`

**E.5. Skeleton-загрузки кастомные** под каждую страницу

**E.6. Локализация**
- KZ / RU / EN
- `next-intl`

### 🟣 ФАЗА F — Интеграции с экосистемой РК (по 1 неделе)

**F.1. Kaspi Pay**
- Kaspi QR — арендатор сканирует QR со счёта-фактуры
- Kaspi Business API — авто-зачисление платежа
- Webhook для уведомления

**F.2. eGov.kz / e-license.kz — проверка БИН**
- Открытое API для проверки контрагента
- Авто-заполнение реквизитов по БИН: название, юр.адрес, директор

**F.3. 1C-Бухгалтерия**
- Экспорт начислений и платежей в формате 1C-EnterpriseData
- Endpoint `/api/export/1c`

**F.4. Krisha.kz / OLX публикация**
- Свободные помещения автоматически публикуются на Krisha
- API Krisha (если откроют) или скрейпер

**F.5. WhatsApp Business**
- Уведомления через WhatsApp (как Telegram)
- Twilio или WATI

**F.6. Sentry**
- Мониторинг ошибок в проде
- Бесплатный tier 5K событий/мес

### ⚫ ФАЗА G — Аналитика для собственника (1 неделя)

**G.1. Дашборд OWNER**
- Большие графики: выручка / расходы / маржа по месяцам (12 мес)
- KPI: ROI, средний срок аренды, заполняемость
- Топ-5 должников
- Топ-5 самых прибыльных помещений

**G.2. Heatmap занятости помещений**
- Цвет ячейки = % времени занято за год
- Видно какие помещения хуже сдаются

**G.3. Когортный анализ арендаторов**
- График: кто пришёл в 2024, кто остался к 2025...
- Удержание (retention)

**G.4. Сравнение тарифов с рынком**
- Парсинг Krisha по похожим помещениям
- Рекомендация «вы продаёте на 15% ниже рынка»

**G.5. Экспорт всех отчётов**
- Excel (exceljs)
- PDF (как договор)
